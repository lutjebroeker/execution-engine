--- WORKFLOW 1: Morning Briefing (cron-morning-briefing) ---

Schedule: 07:00 Europe/Amsterdam, every day

Nodes:

1. **Schedule Trigger**: `0 7 * * *`, timezone Europe/Amsterdam

2. **HTTP Request — Get active tenants**:
   GET `{{ $env.SUPABASE_URL }}/rest/v1/tenants?is_active=eq.true&select=*`
   Headers: service_role key

3. **SplitInBatches**: batchSize=1 (loop one tenant at a time)

4. **HTTP Request — Get today's active goals**:
   GET `{{ $env.SUPABASE_URL }}/rest/v1/goals?tenant_id=eq.{{ $json.id }}&status=eq.active&select=id,title,deadline,progress_pct`

5. **HTTP Request — Get yesterday's daily_priorities** (for the "yesterday" reflection line):
   GET `{{ $env.SUPABASE_URL }}/rest/v1/daily_priorities?tenant_id=eq.{{ $json.id }}&date=eq.{{ $today_minus_1 }}&select=priorities,execution_score`

6. **HTTP Request — Claude API** — generate the morning briefing:
   Use claude-haiku-3-5 (fast, low cost for cron):
   System prompt:
   ```
   You are {{persona_name}}, an accountability coach with {{tone}} tone.
   Generate a morning briefing. Format:
   - Start with "Good morning." (no fluff)
   - "Three things matter today:" followed by exactly 3 numbered priorities selected from active goals
   - Each priority: "[Priority text] — [one-line why it matters to the goal]"
   - Yesterday reflection (if data available): "Yesterday: X/Y completed (Z%)."
   - End: "Anything to add before you start?"
   - Total: 5-8 lines max
   ```
   Message: include active goals list + yesterday score if available.
   Output: text content of briefing.

7. **HTTP Request — Upsert daily_priorities row**:
   POST `{{ $env.SUPABASE_URL }}/rest/v1/daily_priorities`
   Headers: `Prefer: resolution=merge-duplicates`
   Body:
   ```json
   {
     "tenant_id": "{{ $tenant.id }}",
     "date": "{{ $today }}",
     "priorities": [
       {"text": "{{ priority_1 }}", "goal_id": "{{ goal_1_id }}", "completed": null},
       {"text": "{{ priority_2 }}", "goal_id": "{{ goal_2_id }}", "completed": null},
       {"text": "{{ priority_3 }}", "goal_id": "{{ goal_3_id }}", "completed": null}
     ],
     "briefing_sent_at": "{{ $now }}"
   }
   ```
   Note: Extract priorities from Claude's text response using a Code node — parse the numbered list.

8. **HTTP Request — Send Telegram message**:
   POST `https://api.telegram.org/bot{{ $env.TELEGRAM_BOT_TOKEN }}/sendMessage`
   Body: `{ chat_id: {{ $tenant.telegram_chat_id }}, text: "{{ $briefing_text }}", parse_mode: "Markdown" }`

9. **HTTP Request — Save briefing to conversations table**:
   POST `/rest/v1/conversations` with `{tenant_id, role: "assistant", content: briefing_text}`

Error handling: IF Supabase unreachable → log error to n8n execution log, skip tenant, continue loop. Do not crash the entire cron.

--- WORKFLOW 2: Accountability Check-in (cron-accountability-checkin) ---

Schedule: 17:00 Europe/Amsterdam, every day

Nodes:

1. **Schedule Trigger**: `0 17 * * *`, timezone Europe/Amsterdam

2. **HTTP Request — Get active tenants** (same as above)

3. **SplitInBatches**: batchSize=1

4. **HTTP Request — Get today's daily_priorities**:
   GET `{{ $env.SUPABASE_URL }}/rest/v1/daily_priorities?tenant_id=eq.{{ $json.id }}&date=eq.{{ $today }}&select=*`
   IF no row found (briefing didn't fire / user onboarded today): skip this tenant.

5. **Code node — Build check-in message**:
   ```javascript
   const priorities = $json.priorities; // array of {text, completed}
   let msg = "Time to check in.\n\nDid you complete today's priorities?\n\n";
   priorities.forEach((p, i) => {
     msg += `${i+1}. ${p.text}\n`;
   });
   msg += "\nReply with the numbers you completed (e.g. '1 3') or 'all' / 'none'.";
   return [{ json: { check_in_message: msg } }];
   ```

6. **HTTP Request — Send check-in Telegram message**

7. **HTTP Request — Update daily_priorities.checkin_sent_at**:
   PATCH `/rest/v1/daily_priorities?tenant_id=eq.{id}&date=eq.{today}`
   Body: `{ "checkin_sent_at": "{{ $now }}" }`

The user's reply to the check-in comes back through the main bot webhook (telegram-bot-agent.json workflow). The main agent workflow must handle check-in replies:
- Detect if user message is a check-in response (numbers, "all", "none")
- Look up today's daily_priorities for this tenant
- Update each priority's `completed` field
- Calculate execution_score: completed_count / total_count * 100
- PATCH daily_priorities row with completed priorities + execution_score + checkin_completed_at
- Reply: "Got it. X/Y completed today (Z%). [Direct tone: brief acknowledgment or callout if low]"

Add this check-in response detection to the main bot agent workflow (06-02). In the Code node that extracts message data, check if today's checkin_sent_at is set AND checkin_completed_at is null — if so, route to check-in response handler before invoking Claude.

Export both workflows as JSON files.

Build the weekly review workflow. This produces the execution score and pattern observation that are the core proof-of-value of the system (CRON-03).

Schedule: Sunday 18:00 Europe/Amsterdam

Nodes:

1. **Schedule Trigger**: `0 18 * * 0`, timezone Europe/Amsterdam

2. **HTTP Request — Get active tenants**

3. **SplitInBatches**: batchSize=1

4. **Code node — Calculate week boundaries**:
   ```javascript
   const now = new Date();
   const dayOfWeek = now.getDay(); // 0 = Sunday
   const monday = new Date(now);
   monday.setDate(now.getDate() - 6); // last Monday
   const week_start = monday.toISOString().split('T')[0];
   const week_end = now.toISOString().split('T')[0]; // today (Sunday)
   return [{ json: { week_start, week_end } }];
   ```

5. **HTTP Request — Get this week's daily_priorities**:
   GET `{{ $env.SUPABASE_URL }}/rest/v1/daily_priorities?tenant_id=eq.{{ $tenant.id }}&date=gte.{{ $week_start }}&date=lte.{{ $week_end }}&select=date,priorities,execution_score`

6. **HTTP Request — Get last 3 weeks of weekly_reviews** (for pattern detection):
   GET `{{ $env.SUPABASE_URL }}/rest/v1/weekly_reviews?tenant_id=eq.{{ $tenant.id }}&order=week_start.desc&limit=3&select=week_start,execution_score,pattern_observation`

7. **Code node — Calculate weekly execution score**:
   ```javascript
   const days = $input.first().json; // array of daily_priorities rows
   const daysWithScore = days.filter(d => d.execution_score !== null);
   const weeklyScore = daysWithScore.length > 0
     ? daysWithScore.reduce((sum, d) => sum + d.execution_score, 0) / daysWithScore.length
     : null;

   // Identify worst days (days with score < 50%)
   const worstDays = days
     .filter(d => d.execution_score !== null && d.execution_score < 50)
     .map(d => ({ date: d.date, score: d.execution_score, day: new Date(d.date).toLocaleDateString('en-US', {weekday: 'long'}) }));

   return [{ json: { weeklyScore, worstDays, days, daysWithScore: daysWithScore.length } }];
   ```

8. **HTTP Request — Claude API** — generate narrative pattern observation:
   Use claude-haiku-3-5.
   System prompt:
   ```
   You are {{persona_name}} with {{tone}} tone. Analyze this week's execution data and write a pattern observation.
   Rules:
   - One paragraph, 2-3 sentences max
   - If a specific day was consistently low, name it: "Wednesday was your worst day for 3 weeks running"
   - Reference specific priorities that went uncompleted if visible
   - Do NOT list raw numbers — give a narrative insight
   - {{tone}}: direct = call it out plainly, warm = acknowledge effort + pattern, professional = factual observation
   ```
   Input: weekly score, daily breakdown, last 3 weeks of reviews.

9. **HTTP Request — Upsert weekly_reviews row**:
   POST `{{ $env.SUPABASE_URL }}/rest/v1/weekly_reviews`
   Headers: `Prefer: resolution=merge-duplicates`
   Body:
   ```json
   {
     "tenant_id": "{{ $tenant.id }}",
     "week_start": "{{ $week_start }}",
     "execution_score": "{{ $weeklyScore }}",
     "pattern_observation": "{{ $patternText }}",
     "raw_data": {
       "days": "{{ $days }}",
       "days_with_score": "{{ $daysWithScore }}"
     }
   }
   ```

10. **Code node — Build Telegram message**:
    ```javascript
    const score = $json.weeklyScore;
    const scoreLabel = score >= 70 ? 'Strong week.' : score >= 50 ? 'Decent week.' : 'Below target.';
    const scoreDisplay = score !== null ? `${Math.round(score)}%` : 'No data yet';
    const pattern = $json.patternText;

    const msg = `Weekly Review — ${$json.week_start}\n\n` +
      `Execution Score: *${scoreDisplay}* — ${scoreLabel}\n\n` +
      `${pattern}\n\n` +
      `New week starts tomorrow. What's the most important thing to do differently?`;
    return [{ json: { message: msg } }];
    ```

    Score thresholds per CONTEXT.md: 70%+ = good ("Strong week."), 50-70% = okay ("Decent week."), <50% = flagged ("Below target.").

11. **HTTP Request — Send Telegram message**

12. **HTTP Request — Save to conversations table**

Export workflow as `n8n/workflows/cron-weekly-review.json`.
