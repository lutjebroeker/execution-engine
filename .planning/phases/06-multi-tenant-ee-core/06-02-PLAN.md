1. Message @BotFather on Telegram: `/newbot`

4. Save the bot token to /root/.claude/.env as TELEGRAM_BOT_TOKEN

Set the bot webhook to point to n8n:
```bash
curl -s "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/setWebhook" \
  -d "url=https://n8n.jellespek.nl/webhook/telegram-bot"
```
Expected response: `{"ok":true,"result":true}`

STEP B — Build the n8n workflow using n8n MCP tools:

Create workflow named "EE Telegram Bot Agent" with these nodes:

1. **Telegram Trigger** (or Webhook node for manual webhook setup):
   - Path: `telegram-bot`
   - Method: POST
   - Responds immediately with 200 (Telegram requires <5s response)

2. **Code node — Extract message data**:
   ```javascript
   const body = $input.first().json.body;
   const message = body.message || body.edited_message;
   const chat_id = message?.chat?.id;
   const text = message?.text || '';
   const message_id = message?.message_id;
   return [{ json: { chat_id, text, message_id, raw: body } }];
   ```

3. **HTTP Request — Lookup tenant** (Supabase REST):
   - URL: `{{ $env.SUPABASE_URL }}/rest/v1/tenants?telegram_chat_id=eq.{{ $json.chat_id }}&select=*`
   - Headers: `apikey: {{ $env.SUPABASE_SERVICE_ROLE_KEY }}`, `Authorization: Bearer {{ $env.SUPABASE_SERVICE_ROLE_KEY }}`

4. **IF node — Tenant exists?**:
   - Condition: `{{ $json[0]?.id }}` is not empty

5. **HTTP Request — Get last 20 conversations** (TRUE branch):
   - URL: `{{ $env.SUPABASE_URL }}/rest/v1/conversations?tenant_id=eq.{{ $json[0].id }}&order=created_at.desc&limit=20&select=role,content`

6. **Code node — Build Claude messages array**:
   Read system prompt from ee-agent.md content, inject tenant.persona_name and tenant.tone.
   Build messages array: system prompt + last 20 conversation turns (reversed to chronological) + current user message.

7. **HTTP Request — Claude API (first call with tools)**:
   - URL: `https://api.anthropic.com/v1/messages`
   - Method: POST
   - Headers: `x-api-key: {{ $env.ANTHROPIC_API_KEY }}`, `anthropic-version: 2023-06-01`
   - Body: model `claude-opus-4-5`, max_tokens 1024, messages array, tools array (see system prompt)

8. **Code node — Parse Claude response**:
   Check `stop_reason`. If `tool_use`, extract tool name and input. If `end_turn`, extract text content.

9. **Switch node — Route by tool name** (or skip if end_turn):
   Cases: manage_goal, log_progress, get_goals_overview, save_memory, search_memory, web_search, none

10. **Tool execution nodes** — one HTTP Request per tool:
    - manage_goal: POST/PATCH/DELETE to `/rest/v1/goals` based on `action` field (create/update/delete)
    - log_progress: POST to `/rest/v1/goal_logs`
    - get_goals_overview: GET `/rest/v1/goals?tenant_id=eq.{id}&status=eq.active&select=*`
    - save_memory: POST to `/rest/v1/memory` with `{tenant_id, content, tags}`
    - search_memory: GET `/rest/v1/memory?tenant_id=eq.{id}&content=ilike.*{query}*`
    - web_search: GET `https://api.search.brave.com/res/v1/web/search?q={query}` (read BRAVE_API_KEY from .env, or skip if not available — return "web_search not configured")

11. **HTTP Request — Claude API (second call with tool results)**:
    Build messages array including the tool_use and tool_result turns. Call Claude again to generate final text response.

12. **HTTP Request — Send Telegram reply**:
    - URL: `https://api.telegram.org/bot{{ $env.TELEGRAM_BOT_TOKEN }}/sendMessage`
    - Body: `{ chat_id: {{ $json.chat_id }}, text: "{{ $json.response }}", parse_mode: "Markdown" }`

13. **HTTP Request — Save conversation turns** (two calls: user message, then assistant reply):
    POST to `/rest/v1/conversations` with `{tenant_id, role, content}`

FALSE branch (no tenant): handled in Task 2 onboarding flow.

STEP 2: Onboarding state machine (5 steps)
Implement as a Code node reading/writing onboarding state:

```
Step 0 (initial /start):
  Ask: "Hey! I'm your accountability coach. What should I call myself? (e.g. Coach, Alex, Mentor)"

Step 1 (after persona name received):
  Ask: "Got it. What tone do you prefer?
  1. Direct — Alex Hormozi style, no fluff
  2. Warm — Encouraging and personal
  3. Professional — Structured and neutral"

Step 2 (after tone selected):
  Ask: "What's your first 12-week goal? Be specific — what's the outcome you want by [date 12 weeks from now]?"
  (If vague like "be healthier": ask one clarifying question then accept)

Step 3 (after goal 1, optionally collect goal 2):
  Ask: "Any other goals? (up to 2 more, or say 'done' to continue)"

Step 4 (after goals confirmed):
  Ask: "What time do you usually start work? (e.g. 07:30)"

COMPLETION (after work start time):
  Create tenant row in Supabase.
  Create goal row(s) in Supabase.
  Send confirmation:
  "You're live.

Goals set:
• [Goal 1]
• [Goal 2 if provided]
• [Goal 3 if provided]

First briefing: tomorrow at [work_start_time]
Check-in: daily at 17:00

See you then."
```
