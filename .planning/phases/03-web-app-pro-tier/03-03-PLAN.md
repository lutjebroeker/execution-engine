    Each Edge Function follows this security pattern (MUST be in this order — no shortcuts):
    1. Handle OPTIONS → return corsHeaders
    2. Verify user JWT via `supabase.auth.getUser()` (NOT getSession)
    3. Check profiles.subscription_status === 'active' → return 403 if not Pro (defense in depth — client check is UX only)
    4. Fetch user data from Supabase
    5. Call Claude
    6. Return analysis text

    CORS headers for all three:
    ```typescript
    const corsHeaders = {
      'Access-Control-Allow-Origin': '*',
      'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type',
    };
    ```

    **supabase/functions/ai-weekly-reflection/index.ts:**

    Request body: `{ cycleId: string, weekNumber: number }`

    Data fetched: weekly reflection (from weekly_reflections table) + cycle goals (from cycles table)

    Claude prompt (build as a function `buildReflectionPrompt(reflection, goals)`):
    ```
    You are an execution coach reviewing a 12-week cycle weekly reflection.

    Week: {weekNumber} of 12
    Execution score: {completionScore}%

    Goals this cycle:
    {goals.map(g => `- ${g.title}`).join('\n')}

    Weekly reflection:
    What worked well: {whatWorkedWell}
    What didn't work: {whatDidntWork}
    What distracted: {whatDistracted}
    What must change: {whatMustChange}

    Provide a concise (3-5 sentences) coaching analysis: identify the key pattern, name the specific lever to pull next week, and frame it as an honest accountability observation — not cheerleading. Be direct.
    ```

    After getting Claude response: also save `ai_analysis` to the weekly_reflections row using admin client (service role). Return `{ analysis: message.content[0].text }`.

    **supabase/functions/ai-morning-nudge/index.ts:**

    Request body: `{ cycleId: string }`

    Data fetched: active cycle goals from cycles table + today's daily_notes row (morning_checkin if exists) + yesterday's daily_notes row

    Claude prompt (build as `buildMorningNudgePrompt(goals, todayNote, yesterdayNote)`):
    ```
    You are an execution coach giving a morning nudge.

    Active cycle goals:
    {goals.map(g => `- ${g.title}: ${g.tactics.map(t => t.title).join(', ')}`).join('\n')}

    Yesterday: {yesterdayNote?.evening_reflection ? `accomplished: ${yesterdayNote.evening_reflection.accomplished}` : 'no evening check-in'}
    Today's focus (if already set): {todayNote?.morning_checkin?.focusTask || 'not yet set'}

    Write a single short paragraph (2-3 sentences) morning nudge. Reference one specific goal or tactic. Make it feel like a direct message from a coach who knows their system, not a generic motivational quote. No emojis.
    ```

    Return `{ nudge: message.content[0].text }`. Do NOT persist morning nudge (ephemeral — refreshes each day).

    **supabase/functions/ai-cycle-review/index.ts:**

    Request body: `{ cycleId: string }`

    Data fetched: full cycle (from cycles table) + all weekly_reflections for this cycle + twelve_week_reviews row (if exists)

    Claude prompt (build as `buildCycleReviewPrompt(cycle, reflections, review)`):
    ```
    You are an execution coach analyzing a completed 12-week cycle.

    Cycle goals:
    {goals with completion stats}

    Weekly reflection summaries (weeks with data):
    {reflections.map(r => `Week ${r.week_number}: ${r.completion_score}% — ${r.what_worked_well?.substring(0, 100)}`).join('\n')}

    12-week review answers:
    Biggest wins: {review?.biggest_wins}
    Biggest challenges: {review?.biggest_challenges}
    Key lessons: {review?.key_lessons}

    Write a 4-6 sentence cycle analysis: identify the dominant pattern across the 12 weeks, name 1-2 specific growth areas, and give one clear recommendation for the next cycle. Be direct and honest.
    ```

    After getting Claude response: save `ai_analysis` to the twelve_week_reviews row using admin client. Return `{ analysis: message.content[0].text }`.

    **Anti-patterns to avoid:**
    - Never use `getSession()` in Edge Functions — always `getUser()`

    - Never put ANTHROPIC_API_KEY in any src/ file — only `Deno.env.get('ANTHROPIC_API_KEY')`
