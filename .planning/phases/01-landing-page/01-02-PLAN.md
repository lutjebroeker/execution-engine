**Form handler (single function, handles both forms via class selector):**
```javascript
async function handleWaitlistSubmit(e) {
  e.preventDefault();
  const form = e.target;
  const btn = form.querySelector('button[type="submit"]');
  const emailInput = form.querySelector('input[type="email"]');
  const email = emailInput.value.trim();
  const successEl = form.querySelector('.form-success');

  // Prevent double-submit
  if (btn.disabled) return;
  btn.disabled = true;
  btn.textContent = 'Joining...';

  try {
    const { error } = await supabaseClient
      .from('execution_engine_waitlist')
      .insert({ email });

    if (error) {
      // Handle UNIQUE constraint violation (duplicate email)
      if (error.code === '23505') {
        // Show friendly message — not an error, they're already in
        form.innerHTML = '<p class="form-success">You\'re already on the list. Watch your inbox.</p>';
        return;
      }
      throw error;
    }

    // Success: swap form for confirmation message (no redirect per CONTEXT.md)
    form.innerHTML = '<p class="form-success">You\'re on the list. Watch your inbox.</p>';

  } catch (err) {
    console.error('Waitlist error:', err);
    btn.disabled = false;
    btn.textContent = 'Join Waitlist';

    let errEl = form.querySelector('.form-error');
    if (!errEl) {
      errEl = document.createElement('p');
      errEl.className = 'form-error';
      form.appendChild(errEl);
    }
    errEl.textContent = 'Something went wrong. Please try again.';
  }
}

// Attach handler to ALL waitlist forms (hero + bottom CTA)
document.querySelectorAll('.waitlist-form').forEach(form => {
  form.addEventListener('submit', handleWaitlistSubmit);
});
```

**Critical requirements:**
- `btn.disabled = true` fires before async call (prevents double-submit)
- Duplicate email (error code `23505`) gets a friendly "already on the list" message — not an error
- On success: `form.innerHTML` replaces the form (no re-enable)
- On error: button re-enables so user can retry
- No redirect, no page reload on success (CONTEXT.md: inline confirmation only)
- This file uses ESM-style clean code but is NOT an ES module (loaded as regular script — Supabase CDN populates global `supabase`)

**After writing the file:** update `index.html` to replace the Supabase credentials placeholder and the Plausible `data-domain` value. Also update the two OG URL placeholders in `<head>` to use the actual Vercel deployment URL once known (or `executionengine.vercel.app` as placeholder until deploy completes).
