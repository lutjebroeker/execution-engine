  <name>Task 1: Build Gumroad → n8n → Resend welcome email workflow and export as JSON</name>
  <files>n8n-gumroad-welcome-workflow.json</files>
  <action>
    PREREQUISITE CHECK: Before building, verify that Resend has been set up and jellespek.nl domain is verified (user_setup in frontmatter). If the domain is not yet verified in Resend, the workflow will build correctly but sending will fail until verification completes — note this in the workflow description.

    Build the following new n8n workflow (do NOT modify any existing AM workflows):

    **Workflow name:** "Execution Engine — Gumroad Purchase Welcome Email"

    **Node sequence:**

    1. **Webhook node** (trigger)
       - Name: "Gumroad Sale Ping"
       - HTTP Method: POST
       - Path: `gumroad-sale`
       - Response Mode: `responseNode` (respond via dedicated Respond to Webhook node — required for immediate 200 response)

    2. **Respond to Webhook node** (immediately after webhook — CRITICAL for preventing duplicate emails)
       - Name: "Respond 200 OK"
       - Respond With: text
       - Response Body: "OK"
       - Connect directly from Webhook node

    3. **Set node** (extract buyer data)
       - Name: "Extract Buyer Data"
       - Source: `$json.body` (Gumroad sends POST body)
       - Fields to set (all String type):
         - `buyerEmail` = `={{$json.body.email}}`
         - `productName` = `={{$json.body.product_name}}`
         - `orderNumber` = `={{$json.body.order_number}}`
         - `licenseKey` = `={{$json.body.license_key}}`
       - Connect from Respond to Webhook node (parallel branch, not sequential — both run after webhook fires)
       - Note: In n8n, the Webhook can connect to both Respond to Webhook AND the Set node simultaneously. The Set node chain runs after the immediate 200 response.

    4. **HTTP Request node** (send welcome email via Resend)
       - Name: "Send Welcome Email via Resend"
       - Method: POST
       - URL: `https://api.resend.com/emails`
       - Authentication: Generic Credential Type → HTTP Header Auth
       - Header Auth credential: name = `Authorization`, value = `Bearer YOUR_RESEND_API_KEY` (buyer replaces with real key, or store in n8n credential)
       - Send Body: true
       - Content Type: JSON
       - Body (JSON):
         ```json
         {
           "from": "Jelle Spek <support@jellespek.nl>",
           "to": ["={{$json.buyerEmail}}"],
           "subject": "Your Execution Engine AM Bundle — Start Here",
           "html": "[welcome email HTML — see below]"
         }
         ```

    **Welcome email HTML body:**
    Keep it clean and readable. Structure:
    - H1: "Welcome to Execution Engine"
    - Intro paragraph: "You've got everything you need to go from zero to your first Telegram check-in in 30 minutes."
    - H2: "Start Here — Your Checklist"
    - Numbered list (5 items): 1) Open README.md in the bundle, 2) Install prerequisites (n8n, Telegram, Obsidian, Ollama, PostgreSQL), 3) Follow the 7-step setup guide, 4) Activate your workflows, 5) Send /start to your Telegram bot
    - Download reminder: "Your bundle download link is in your Gumroad receipt email. If you can't find it, visit https://app.gumroad.com/library"
    - Support line: "Any questions? Email support@jellespek.nl — I reply within 24 hours."
    - Sign-off: "— Jelle"

    Connect Set node → HTTP Request node.

    After building the workflow in n8n:
    - Export this single workflow as JSON: n8n UI → open workflow → ⋮ menu → Download
    - Save as `n8n-gumroad-welcome-workflow.json` at the project root (NOT inside the bundle directory — this is for reference/backup, not for buyer delivery)

    The n8n Webhook URL for this workflow (shown in the Webhook node settings) is what gets entered in Gumroad Dashboard → Settings → Advanced → Ping a URL. Note it down for the human checkpoint.
