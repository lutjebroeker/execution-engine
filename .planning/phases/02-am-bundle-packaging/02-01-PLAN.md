<task type="auto">
  <name>Task 2: Refactor all workflows to Config node pattern and export bundle JSON</name>
  <files>bundle/workflows/am-workflows-v1.json</files>
  <action>
    Read `bundle/audit-notes.md` first to understand which workflows need Config node changes and which values to centralise.

    For each workflow that contains hardcoded personal values (per audit-notes.md Section 2):
    1. Open the workflow in n8n
    2. Add a Set node named exactly "CONFIG — Edit These Values" immediately after the trigger (before any other processing node)
    3. Add all buyer-configurable fields to the Set node using the naming convention from RESEARCH.md (see interfaces block above). Replace real values with placeholder strings (e.g. "YOUR_TELEGRAM_CHAT_ID")
    4. Update every downstream node that previously used a hardcoded value to reference the Config node via expression: `{{$node["CONFIG — Edit These Values"].json.telegramChatId}}`
    5. For Postgres credential nodes: credentials are handled by n8n credential system — the Config node covers connection details only if they are set directly in node parameters. If using proper n8n Postgres credentials (recommended), ensure the credential name in exported JSON is set to a generic placeholder like "Postgres — configure credentials on import"
    6. Save each workflow after refactoring

    If a workflow has multiple triggers (e.g. Schedule + Manual), both trigger branches must feed through the Config Set node before any node that uses configured values. Use a Merge node if needed to keep the Config node single.

    After all workflows are refactored:
    - Run: `n8n export:workflow --all --output=bundle/workflows/am-workflows-v1.json`
    - If CLI is not available: In n8n UI, Workflows list → select all → Actions → Download → save as `bundle/workflows/am-workflows-v1.json`

    Create the `bundle/workflows/` directory if it does not exist.

    After export, open the JSON file and do a quick scan: grep for any of Jelle's personal values (Telegram chat ID, actual Obsidian URL, Ollama URL with non-placeholder content, actual Postgres passwords). If found, fix in n8n and re-export.

    Community nodes from audit-notes.md Section 1: if any community node is used, note it in the exported file's README (handled in Plan 02-02) and add it to the prerequisites. Do NOT refactor away from community nodes unless they block a clean install — just document the install requirement.
  </action>
